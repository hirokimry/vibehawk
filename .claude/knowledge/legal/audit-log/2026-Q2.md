### 法務観点評価

**Mission「レビューツールに追加課金が要らない世界をつくる」**
- 商標リスク: 既存製品名（CodeRabbit / GitHub / Greptile 等）を含まず、特定製品への誹謗・誤認を誘発する文言なし。
- 反競争的表現: 機能訴求であり、競合製品への虚偽広告・誹謗には該当しない。
- 判定: 問題なし。

**Value 1「利用者の契約だけで、完結させる」**
- LLM プロバイダー等の「再販・代理利用禁止」条項の対象になりうる設計を排除する思想。
- vibehawk が利用者の API キーを横取りしない設計の法務上の根拠となる。

**Value 2「観察する、書き換えない」**
- GitHub API の書き込み権限濫用リスクを設計レベルで排除。
- PR メタデータの読み取りのみ行い書き換えを行わない制約は ToS 遵守上の重要な設計制約。

**Value 4「公式の道を、迂回せず歩く」**
- スクレイピング・非公式エンドポイント禁止を Value として宣言。
- GitHub / Anthropic / その他外部サービスの ToS 遵守を設計思想レベルで担保。
- legal-principles.md の「外部サービスの利用規約に明示的に違反する変更 → 即ブロック」の判断基準の根拠となる。
