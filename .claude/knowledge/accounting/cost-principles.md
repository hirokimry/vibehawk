# コスト判断原則

docs/cost-analysis.md のコスト構造と MVV.md から導出される、経理分析員の判断基準。
コスト分析の詳細は `docs/cost-analysis.md` を参照すること。

## コスト構造の前提

vibehawk のコスト構造は MVV から規定される。詳細は `docs/cost-analysis.md` の「vibehawk コスト設計原則（MVV 由来）」セクションを参照。

### 利用者側コスト（MUST NOT）

vibehawk は以下を利用者に課してはならない:

- 専用 SaaS の追加月額（CodeRabbit Pro / Greptile 等との対比優位がここにある）
- 専用 LLM API キーの発行・管理（Anthropic API キー直接利用 等）
- 自前サーバーのホスティング費用

**設計原則**: 利用者が既に契約している LLM プロバイダー（Claude Pro / Max 等）の OAuth 枠内で完結する。

### 固定費

vibehawk 運営側の固定費は最小化を目指す。公式 OAuth 経由の設計は外部 API の従量コストを振らない構造になっている。

### 変動費

利用者数増加が vibehawk 運営側の従量コスト増加に直結するアーキテクチャは採用しない。
