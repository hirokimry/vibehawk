# 経理監査ログ 2026-Q2

## 2026-05-08 — MVV 制定によるコスト設計原則の確立

### 対象

vibehawk MVV 制定（Mission / Value 1）によるコスト思想の `docs/cost-analysis.md` および `cost-principles.md` への反映

### 判断内容

MVV 制定により以下のコスト設計原則を確立した。

**利用者側コスト（MUST NOT）**

- 専用 SaaS 月額（CodeRabbit Pro / Greptile 等）を要求する設計を採用しない
- 専用 LLM API キーの発行を要求する設計を採用しない
- 自前サーバーのホスティング費用を要求する設計を採用しない

**アーキテクチャ原則**

- 利用者が既に契約している LLM プロバイダー（Claude Pro / Max 等）の公式 OAuth 枠内で完結する
- 利用者数増加が vibehawk 運営側の従量コスト増加に直結するアーキテクチャを採用しない

### 根拠

- Mission「レビューツールに追加課金が要らない世界をつくる」
- Value 1「利用者の契約だけで、完結させる — 自前のサーバーも追加 API キーも持たず、利用者が既に結んでいる契約の内側で完結する」
- Value 4「公式の道を、迂回せず歩く」（非公式 API・スクレイピングによるコスト回避策を選ばない）

### 反映先

- `docs/cost-analysis.md` — 「vibehawk コスト設計原則（MVV 由来）」セクション新規追加
- `.claude/knowledge/accounting/cost-principles.md` — コスト構造の前提セクションを MVV 基準で埋めた

### スケール観点

利用者数が増加しても vibehawk 運営側のコスト構造は増加しない（計算リソースは利用者側 LLM プロバイダーが負担）。これは競合 SaaS（CodeRabbit 等）に対するコスト構造上の根本的な差別化点であり、スケール耐性が高い。

### 判定

反映完了。MVV 由来の原則のため変更不可（ファウンダーのみ変更可）。
