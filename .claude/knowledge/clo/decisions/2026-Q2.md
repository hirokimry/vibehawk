# CLO 判断記録 2026-Q2

## 2026-05-10 — Issue #91 `@clack/prompts` 依存追加 CLO 承認

### 判断種別

条件付き承認

### 対象

`@clack/prompts` v1.3.0（bombshell-dev/clack）を `dependencies` として追加する計画（Issue #91 setup wizard）。

### 合議状況

- 法務チーム（plan-legal）3回実行の合議結果: 全員一致（3件の指摘を提示、いずれも「リスクあり」ではなく「手続き不備・ケア要請」レベル）
- 全会一致ルール: 3名全員が手続き上の懸念を示したが、ライセンス本体の拒否判定はなし
- ただし指摘 #1「CLO 合議未完了」は CLO メタレビュー（本判断）をもって解消

### メタレビュー判定

#### ライセンス整合

- `@clack/prompts` ライセンス: MIT（npm registry にてライブ確認済み、v1.3.0）
- vibehawk ライセンス: MIT
- `docs/POLICY.md` L101: 「MIT / BSD / Apache 2.0 は原則許可」
- 結論: 問題なし

#### POLICY.md 整合

- 平社員層指摘 #3 の通り、README 集約時に以下の必須維持項目が欠落するリスクがある
  - Anthropic 送信通知（`docs/POLICY.md` L111-119）
  - CLI 非書込み宣言（`docs/POLICY.md` L122-126、L243-247）
  - 追加課金ゼロの適用範囲・対象外ケース
- 計画の Phase 3「必須維持項目」明記とPOLICY.md整合チェックで対応する方針: 計画文書上は手当て済み
- ただし実装段階での実際の整合チェックが担保されるまで条件付き

#### 利用者伝搬経路評価

- `@clack/prompts` は `npx vibehawk setup` の CLI UI 層にのみ使用
- 利用者が自リポジトリに配置する workflow（`.github/workflows/vibehawk-review.yml`）には含まれない
- よって利用者の workflow 実行時（PR レビュー実行時）に `@clack/prompts` が利用者の処理系に埋め込まれることはない
- 評価「利用者伝搬: CLI ツール用途のため利用者の workflow 実行時には埋め込まれない」は妥当

#### npm AUP 遵守（`docs/POLICY.md` L281-291）

- `npx vibehawk setup` は対話型ウィザードであり、`[Y/n]` 明示的同意プロンプトを取得する設計
- `--dry-run` フラグで予行演習モードを提供する設計
- 計画 Phase 3 テスト項目に `setup --dry-run` 出力検証あり
- `@clack/prompts` 導入自体が npm AUP 違反を引き起こす要因はない
- 問題なし

#### 代替案比較

| パッケージ | ライセンス | サイズ | 評価 |
|-----------|-----------|--------|------|
| `@clack/prompts` v1.3.0 | MIT | 4KB gzip | TS-native、`group()` API がウィザード向き、2026年定番 |
| `@inquirer/prompts` | MIT | 比較的大きい | 実績豊富だがウィザード向けグループAPIが弱い |
| `enquirer` | MIT | やや大きい | 更新頻度低め、メンテナンスリスクあり |

計画上の選定根拠（4KB / TS-native / `group()` API）は妥当。代替案選択を強制すべき法的理由はない。

#### メンテナンス状況

- bombshell-dev/clack は活発なOSS（2026年5月時点）
- ライセンス変更履歴: MIT を継続維持（npmレジストリ確認）
- 月次リスク監視対象（`docs/POLICY.md` L79-97、`docs/external-dependency-audit.md`）への追加を条件とする

#### 見落としのチェック

- MIT ライセンスの著作権表示義務: vibehawk が再配布する際に `@clack/prompts` の COPYRIGHT を含める義務。vibehawk は CLI として npm 配布するため、`node_modules` に同梱される形で自動的に LICENSE ファイルが含まれる。個別の表示義務追加は不要
- サプライチェーン汚染リスク: `@clack/prompts` の依存ツリーを確認する手順が計画に含まれていない。`npm ls @clack/prompts --all` または `npm audit` をPhase 1 実施時に実行することを条件とする
- 過去判断との一貫性: 本判断が初記録（decisions.md / decisions-index.md 両方とも初回）。外部依存に関する先行判断として `docs/external-dependency-audit.md` の claude-code-action 判断（MIT、問題なし）があり、本判断と整合する

### 最終判断

**条件付き承認**

`@clack/prompts` v1.3.0 の `dependencies` 追加を以下の条件のもとで承認する:

1. **Phase 1 実施時に `npm audit` および `npm ls @clack/prompts --all` を実行**し、推移的依存ライセンス（transitive dependencies）に GPL 系・商用禁止ライセンスが含まれないことを確認すること
2. **`docs/external-dependency-audit.md` への監査エントリ追加**（計画 Phase 1 に明記済み）を必ず実行すること
3. **README 集約時に Anthropic 送信通知・CLI 非書込み宣言・追加課金ゼロの適用範囲の 3 項目**が欠落しないこと（計画 Phase 3 の POLICY.md 整合チェックで PR 本文への確認結果記載を要件とする）
4. **月次リスク監視対象**（`docs/external-dependency-audit.md` 5-1 の監視対象テーブル）に `@clack/prompts` を追加すること（ライセンス変更・破棄リスクの継続監視）

上記 4 条件を PR 本文および `docs/external-dependency-audit.md` の監査エントリで明示することを求める。CLO として最終的な法的判断は人間（CEO/CLO 相当の担当者）が確認の上で確定すること。
