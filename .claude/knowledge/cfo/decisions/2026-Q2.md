### コスト計算の検証

閾値設計の根拠:
- CodeRabbit 互換閾値（30/80/3000）を採用しているため、利用者の既存感覚と一致する
- `3000超 → サマリのみ` の安全弁は、claude-code-action の単一リクエストコスト爆発を防ぐ合理的な上限
- 閾値はすべて `.vibehawk.yaml` でオーバーライド可能なため、大規模モノレポ利用者が追加コストを意図的に受け入れる選択肢が確保されている

### スケール観点

- 並列 PR 実行コストは利用者 workflow の `concurrency:` に委譲済みのため、ツール側のコスト増分は PR サイズにのみ依存する
- 利用者が払う LLM コストは「利用者が既に結んでいる契約の内側で完結」（Value 1）に整合
- ツール側（vibehawk 開発側）に追加 API コストは発生しない構造

### 前提条件

- claude-code-action の リトライ機構が LLM rate limit を吸収する前提（依存）
- `gh api` のバックオフが GitHub API rate limit を吸収する前提（依存）
- 上記 2 点はツール外の仕様であり、変更時は本閾値設計の再評価が必要
