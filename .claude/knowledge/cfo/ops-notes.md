# CFO 運用ノウハウ

CFO メタレビュー時に観察された再発しうるパターンと対処法を蓄積する。
decisions/ が「個別判断の記録」なのに対し、本ファイルは「横断的な運用知見」を扱う。

## 1. docs-only 差分でも合議が機械トリガーされる

**観察日**: 2026-06-12（PR #345、Issue #344）

### 現象

`README.md` のみの再構成差分（intent/docs、挙動不変）に対し、
accounting-analyst x3 + CFO メタレビューの合議フルフローが起動した。

### 原因

diff キーワードマッチャー（`npx` / `ANTHROPIC_API_KEY` / `従量` / `claude -p`）が
README 本文の言及でもヒットするため、ドキュメント再構成 PR でも合議トリガーされる。

### コスト影響

Claude Max 枠内の利用であれば追加課金ゼロ。
今回は Max 枠内で完結（docs/cost-analysis.md の「LLM コストは利用者 Pro/Max 枠に帰属」と整合）。

### 対処指針

- docs-only PR でも合議結果が「全員一致・問題なし」であれば正常動作。
- 合議フルフローが起動すること自体は誤検知ではなく仕様どおり（保全漏れを防ぐ保守的設計）。
- CFO メタレビューでは「docs-only 差分の合議」と認識したうえで、
  コスト計算の影響なし・課金記述の保全可否を確認すれば十分。
- フローを短縮したい場合は、diff キーワードマッチャーの除外リスト追加を
  CPO / CTO に提案する（CFO の管轄外）。

## 2. README 改稿時の課金記述 grep 保全対象

**確立日**: 2026-06-12（PR #345、Issue #344）

### 背景

README には「追加課金が発生するケース / しないケース」の区別文が含まれる。
この区別文は免責の核心であり、改稿時に意味が変わっていないかを保全検証する必要がある。

### 保全対象キーワード（accounting-analyst が独立検証すべき記述）

| キーワード / フレーズ | 保全の趣旨 |
|----------------------|-----------|
| 追加課金が発生しない / 追加課金ゼロ | 課金ゼロ条件の主張が維持されているか |
| 追加課金が発生する / 従量課金 | 課金発生条件の記述が正確に残っているか |
| サポート対象外 | API Key 利用者向けの免責が残っているか |
| ANTHROPIC_API_KEY | fail-fast 挙動（追加課金ゼロ保証）の言及が残っているか |
| npx vibehawk review | 二重課金注意文が残っているか |
| Pro / Max | 枠内完結の主張が残っているか |

### 検証手順

1. 改稿前後の diff で上記キーワードを含む行を抽出する。
2. 削除・変更された行がある場合、同義の記述が別の箇所に存在するかを確認する。
3. 同義保全が確認できれば問題なし。記述ごと消えていれば差し戻し対象。

### 参照先

詳細な数値・条件定義は `docs/cost-analysis.md` が Source of Truth。
README は cost-analysis.md の要約であるため、齟齬が生じた場合は cost-analysis.md を優先する。
